# RootPilot Project Context

## Project Identity

Name:
RootPilot

Tagline:
Autonomous Incident Investigation Platform

---

## Purpose

RootPilot is a portfolio-grade engineering project focused on:

* AI infrastructure
* observability
* distributed systems
* autonomous investigation workflows

The project is intended to demonstrate practical AI engineering skills combined with backend and cloud-native architecture expertise.

---

## Primary Objectives

* Learn AI engineering through practical implementation
* Build a production-style platform
* Demonstrate distributed systems knowledge
* Showcase microservice architecture skills
* Build a strong open-source portfolio project

---

## Technical Direction

The platform will prioritize:

* modular architecture
* provider abstraction
* event-driven communication
* observability-first design
* clean engineering practices

Infrastructure providers must remain replaceable.

Examples:

* RabbitMQ → Kafka
* OpenAI → Anthropic/Ollama
* ElasticSearch → Loki

---

## Engineering Philosophy

RootPilot is intentionally designed as:

* infrastructure-oriented
* backend-heavy
* AI-assisted
* production-minded

The project avoids becoming:

* a toy chatbot
* a simple RAG demo
* a frontend-first product

---

## Development Strategy

The project will evolve incrementally.

Initial focus:

* architecture
* abstractions
* ingestion
* investigation workflows

UI and enterprise concerns are secondary.

---

## Initial MVP Scope

Core MVP capabilities:

* telemetry ingestion
* event correlation
* AI investigation workflows
* RCA generation
* asynchronous service communication

---

## Long-Term Vision

RootPilot may eventually evolve into:

* AI-native observability tooling
* SRE investigation assistant
* cloud diagnostics platform
* autonomous operational intelligence system

---

## Important Constraints

* Avoid proprietary/company-specific implementations
* Avoid overengineering early stages
* Maintain provider-agnostic design
* Focus on practical engineering value
* Optimize for learning + portfolio quality
